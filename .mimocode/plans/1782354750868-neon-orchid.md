# Fix PdfExamView: Always side-by-side layout + prevent PDF download

## Problem
1. On mobile, PDF viewer stacks vertically and doesn't display (iframe + GitHub Releases forces download)
2. User wants: left=PDF, right=answers+audio — always side-by-side, never download PDF

## Root Cause
- GitHub Releases serves PDF with `Content-Disposition: attachment` → browser downloads instead of displaying inline
- Current responsive layout uses `flex-col lg:flex-row` → stacks on mobile

## Plan

### 1. Replace iframe with pdf.js canvas renderer
- Install `pdfjs-dist` package
- Create a `PdfViewer` component that renders PDF pages to `<canvas>` elements
- Load PDF from URL via pdf.js `getDocument()` → renders inline, no download
- Add page navigation (prev/next) and scroll

**Files:**
- `src/components/PdfViewer.tsx` (new)
- `package.json` (add pdfjs-dist)

### 2. Fix PdfExamView layout — always side-by-side
- Change `flex-col lg:flex-row` → always `flex-row`
- Left panel: PdfViewer (60% width desktop, scrollable)
- Right panel: audio + bubble sheet (40% width desktop, scrollable)
- On mobile: use horizontal scroll or tab toggle between PDF/answers

**File:** `src/components/PdfExamView.tsx`

### 3. Handle mobile side-by-side
- Desktop (lg+): 60/40 split, both visible
- Mobile (<lg): Tab toggle between "De thi" and "Dap an" — only one panel visible at a time
- This avoids cramped side-by-side on small screens while keeping the split-view feel

**File:** `src/components/PdfExamView.tsx`

## Verification
1. `npx tsc --noEmit` — no errors
2. `npx vitest run` — tests pass
3. Manual: open `/pdf-exam/toeic-2024-pdf-1` on desktop → PDF left, answers right
4. Manual: open on mobile → tab toggle works, PDF renders inline (no download)
5. `curl -I` on PDF URL still returns attachment header, but pdf.js ignores it
